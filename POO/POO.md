# Object-Oriented Programming (Programação Orientada a Objetos)

## Definição

**Programação Orientada a Objetos (POO)** é um **paradigma de programação** que organiza sistemas de software a partir de **objetos**, entidades que encapsulam **dados** e **comportamentos** relacionados.

Formalmente:
> POO estrutura o código como um conjunto de objetos que interagem por meio de métodos bem definidos, promovendo organização, reutilização e manutenção do software.

Esse paradigma é amplamente adotado na indústria de software por facilitar:
- desenvolvimento de sistemas escaláveis
- manutenção de código a longo prazo
- trabalho em equipe
- evolução contínua de sistemas

---

## Motivação no Mercado de Software

Em sistemas reais:
- o código cresce continuamente
- múltiplos desenvolvedores trabalham em paralelo
- requisitos mudam com frequência

A POO surge como uma resposta a esses desafios ao fornecer:
- separação clara de responsabilidades
- modularização do código
- contratos explícitos entre componentes

Esses fatores tornam a POO dominante em:
- sistemas corporativos
- aplicações web
- APIs
- sistemas distribuídos
- software de larga escala

---

## Conceitos Fundamentais

A POO é construída sobre um conjunto reduzido de conceitos centrais:

- classe
- objeto
- atributo
- método
- estado
- identidade

Esses conceitos formam a base para padrões, arquiteturas e frameworks modernos.

---

## Classe

### Definição

Uma **classe** é uma **estrutura de software** que define:
- quais dados um objeto possui
- quais operações podem ser executadas sobre esses dados

Formalmente:
> Uma classe funciona como um **contrato** que descreve a estrutura e o comportamento de seus objetos.

No contexto de mercado, classes representam:
- entidades de negócio
- componentes de sistema
- modelos de domínio

---

## Objeto

### Definição

Um **objeto** é uma **instância concreta de uma classe**, criada em tempo de execução.

Formalmente:
> Um objeto combina dados e comportamentos e possui uma identidade única dentro do sistema.

Objetos são as unidades fundamentais de execução em sistemas orientados a objetos.

---

## Atributos

### Definição

**Atributos** são os **dados internos** de um objeto.

Eles representam:
- informações persistentes
- estados temporários
- configurações internas

Atributos não devem ser acessados diretamente sem controle, pois isso compromete a integridade do sistema.

---

## Métodos

### Definição

**Métodos** são as **operações** que definem o comportamento de um objeto.

Formalmente:
> Um método descreve como um objeto responde a uma mensagem ou requisição.

No mercado de software, métodos representam:
- regras de negócio
- operações de serviço
- transformações de dados

---

## Estado

### Definição

O **estado** de um objeto corresponde aos valores atuais de seus atributos em um determinado momento da execução.

Mudanças de estado ocorrem exclusivamente por meio de métodos, garantindo previsibilidade e controle.

---

## Identidade

### Definição

A **identidade** é a característica que distingue um objeto de todos os outros objetos do sistema.

Mesmo que dois objetos possuam os mesmos dados, eles são considerados distintos se ocupam posições diferentes na memória ou representam entidades diferentes no domínio do sistema.

---

## Encapsulamento

### Definição

**Encapsulamento** é o princípio que determina que os detalhes internos de um objeto devem ser protegidos contra acesso externo direto.

Formalmente:
> O encapsulamento impõe uma separação clara entre **o que o objeto faz** e **como ele faz**.

No mercado, esse princípio é essencial para:
- reduzir impacto de mudanças
- facilitar refatorações
- garantir estabilidade de APIs

---

## Interface Pública

### Definição

A **interface pública** de uma classe é o conjunto de métodos expostos para interação com outros componentes.

Ela define um **contrato de uso**, permitindo que implementações internas mudem sem afetar o restante do sistema.

---

## Benefícios Práticos da POO no Mercado

A adoção da POO possibilita:
- código mais legível
- reutilização de componentes
- testes automatizados mais simples
- manutenção sustentável
- integração com frameworks e bibliotecas modernas

Esses fatores explicam por que a POO é um requisito central em:
- vagas de backend
- engenharia de software
- arquitetura de sistemas
- desenvolvimento de APIs

---

## Observação Final

A Programação Orientada a Objetos não é apenas uma técnica de implementação, mas uma **estratégia de organização de software** adotada amplamente pela indústria para lidar com sistemas complexos e em constante evolução.
